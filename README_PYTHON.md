# Polymarket 自动交易系统 (Python版本)

基于Python的Polymarket平台自动交易脚本，支持真实API交易，带有详细的实时交易仪表盘。

## 功能特性

- ✅ 自动5分钟交易周期
- ✅ 双向挂单策略（同时挂多空单，一侧成交后取消另一侧）
- ✅ 止损止盈功能
- ✅ 真实交易模式（使用真实API）
- ✅ **详细的实时交易仪表盘**
  - 市场信息面板（价格、订单簿、价差）
  - 账户信息面板（余额、盈亏）
  - 交易统计面板（胜率、总盈亏）
  - 当前持仓面板（实时盈亏）
  - 挂单状态面板
  - 交易历史面板
  - 系统状态面板
- ✅ 实时日志输出
- ✅ 交易历史记录
- ✅ 仓位自动计算（根据余额动态调整）
- ✅ 配置持久化

## 交易策略

### Polymarket 机制说明

**重要**：Polymarket 的 YES 和 NO 是**两个不同的代币**，不是同一代币的多空对冲！

- **YES 代币**：表示"同意"某个预测
- **NO 代币**：表示"不同意"某个预测

**关键关系**：YES 价格 + NO 价格 = 100

### 交易策略

**核心逻辑**：同时买入 YES 和 NO @ 75，随机选择一侧成交，然后监控止损止盈

**注意**：
- 所有操作都是**做多**，没有做空
- YES 和 NO 是两个不同的代币

### 仓位计算规则

```
初始余额：120
初始开仓：10

当 余额 ≥ 初始余额 × 3^n 时
  开仓金额 = 初始开仓 × 2^n（出现小数取整）

其中 n = 0, 1, 2, 3...
```

### 交易流程

1. 每5分钟一个周期
2. 从Polymarket读取当前余额
3. 计算开仓金额
4. **同时挂 YES 买单和 NO 买单 @ 75**（两个不同的代币）
5. 真实API等待成交
6. 取消未成交的一侧
7. 监控价格触发（止损 @45，止盈 @95）
8. 如果到期（5分钟）未触发，按照事件结果结算：
   - 从 Polymarket API 获取事件结果
   - 持仓代币获胜 → 平仓价 = 100
   - 持仓代币失败 → 平仓价 = 0
9. 计算盈亏，更新余额
10. 记录交易

### 止损止盈

**统一的止损止盈逻辑**（无论 YES 还是 NO）：

- **止损**：价格 ≤ 45（止损价）
- **止盈**：价格 ≥ 95（止盈价）

**为什么是这个逻辑？**

1. **做多 YES @ 75**：
   - 止损：YES 跌到 45 → 亏损约 40%
   - 止盈：YES 涨到 95 → 盈利约 27%

2. **做多 NO @ 75**：
   - 止损：NO 跌到 45（意味着 YES 涨到 55）→ 亏损约 40%
   - 止盈：NO 涨到 95（意味着 YES 跌到 5）→ 盈利约 27%

3. **统一性**：无论做多 YES 还是 NO，止损止盈逻辑都一样

### 到期结算

**重要**：如果在 5 分钟交易周期内没有触发止损或止盈，系统将按照事件结果结算：

**事件结果机制**：
- Polymarket 的每个预测事件最终会有一个结果（YES 或 NO）
- 如果事件结果为 YES，YES 代币价值 = 100，NO 代币价值 = 0
- 如果事件结果为 NO，YES 代币价值 = 0，NO 代币价值 = 100

**到期结算逻辑**：
- 从 Polymarket API 获取结算结果（需要事件已结算）
- **结算规则**：
  - 持仓代币获胜 → 平仓价 = 100 → 盈利
  - 持仓代币失败 → 平仓价 = 0 → 亏损

**示例**：
- 场景 1：持仓 YES @ 75，事件结果 = YES → 平仓价 = 100，盈利 = (100-75) × 10 / 75 = $3.33
- 场景 2：持仓 YES @ 75，事件结果 = NO → 平仓价 = 0，亏损 = (0-75) × 10 / 75 = -$10.00

## 真实API模式

### 身份验证说明

Polymarket CLOB API 使用两级身份验证：

- **L1 身份验证（私钥）**: 用于创建 API 凭证、派生凭证、本地签名
- **L2 身份验证（API 凭证）**: 用于交易操作（下单、撤单、查询余额）

**公开 API 无需身份验证**：
- 获取市场列表
- 获取价格
- 获取订单簿

**私有 API 需要身份验证**：
- 创建订单
- 取消订单
- 查询余额
- 查询未成交订单

### 准备工作

#### 1. 获取钱包私钥

- 使用 MetaMask 钱包
- 账户详情 → 导出私钥
- 安全复制私钥（请勿泄露给任何人）

#### 2. 获取 API 凭证

**方法 1: 自动创建（推荐）**

程序可以自动创建 API 凭证：

```python
from polymarket_api import PolymarketClient

client = PolymarketClient(private_key="0x...")
credentials = client.create_api_credentials()
# 返回: {"apiKey": "...", "secret": "...", "passphrase": "..."}
```

**方法 2: 手动配置**

如果已有 API 凭证，可以直接配置：
- API Key
- API Secret
- Passphrase

#### 3. 获取 Funder Address

- 访问 https://polymarket.com/settings
- 查看 Polymarket 个人资料地址
- 复制代理钱包地址（funder 地址）

#### 4. 确定签名类型

| 签名类型 | 值 | 说明 |
|---------|---|------|
| EOA | 0 | 标准 Ethereum 钱包（MetaMask） |
| POLY_PROXY | 1 | Magic Link 代理钱包 |
| GNOSIS_SAFE | 2 | Gnosis Safe 多签代理钱包（最常见） |

大多数用户使用 **GNOSIS_SAFE (2)**。

### 在交互菜单中配置

1. 运行程序: `python main.py`
2. 选择"3. 配置API凭证"
3. 输入以下信息：
   - **Private Key**（必需，用于真实交易）
   - **API Key**（可选，留空则自动创建）
   - **API Secret**（可选，留空则自动创建）

   - **Signature Type**（默认 2）
   - **Funder Address**（从 Polymarket.com 获取）
4. 切换到真实 API 模式

### API 优化说明

程序已针对 Polymarket API 速率限制进行全面优化：

### 速率限制保护

自动遵守所有 API 速率限制：

- Gamma API：300 req / 10s
- CLOB API：1,500 req / 10s
- 交易 API：3,500 req / 10s（突发）

当接近速率限制时，程序会自动等待：

```
⏳ 速率限制：等待 10.10 秒...
```

### 智能缓存

永久缓存不变数据，避免重复获取：

- `token_ids_cache`：市场代币 ID
- `market_details_cache`：市场详情

缓存命中率通常 > 60%。

### 性能优化

- **优化查询**：直接查询单个市场，而非获取全部市场列表
  - 从获取 1000+ 条记录 → 获取 1 条记录
  - 减少 99.9% 的数据传输量
- **批量请求**：使用批量 API 减少调用次数
- **API 统计**：实时监控 API 使用情况

## 常见问题

**Q: 是否需要 API 凭证？**
A: 公开 API 不需要，私有 API 需要。程序可以自动创建。

**Q: 如何选择 Signature Type？**
A: 大多数用户使用 GNOSIS_SAFE (2)。如果是标准 MetaMask 钱包使用 EOA (0)。

**Q: Funder Address 从哪里获取？**
A: 访问 https://polymarket.com/settings 查看你的 Polymarket 个人资料地址。

**Q: API 凭证丢失怎么办？**
A: 使用相同的 nonce 调用派生函数，或创建新的凭证。
